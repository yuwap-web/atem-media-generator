## 📋 解決した問題

### 問題1: テキスト入力時に「1文字入れるたびに入れ直さなければならない」
**原因**: パラメータ変更時にリアルタイム自動プレビュー更新が、毎文字ごとに発火していた

**解決策**: デバウンス機構を実装
- 入力中は500msの間、プレビュー更新を遅延させる
- ユーザーが入力を続けると、タイマーはリセットされる
- 入力が止まって500ms経つと、はじめてプレビューが更新される
- **結果**: スムーズな入力体験で、プレビューは自動更新される

### 問題2: フォントを選んでも日本語が入力できない
**原因**: PyQt5のQLineEditで日本語IME対応が不足していた

**解決策**: 入力メソッド対応を強化
- `Qt.WA_InputMethodEnabled` 属性をすべての入力フィールドに追加
- macOS、Windows、Linuxの全プラットフォームでIME互換性が向上
- **結果**: 日本語、中国語、韓国語など多言語テキスト入力が正常に動作

## 🔧 技術実装詳細

### デバウンス機構
```python
# config.py に追加
PARAMETER_INPUT_DEBOUNCE_MS = 500  # ミリ秒

# ui/parameter_editor.py / ui/advanced_editor.py に実装
self.debounce_timer = QTimer()
self.debounce_timer.setSingleShot(True)
self.debounce_timer.timeout.connect(self.on_text_change_debounced)

# テキスト変更時
def on_parameter_text_changed_debounce(self):
    self.debounce_timer.stop()
    self.debounce_timer.start(self.debounce_delay)  # 500ms

# 500ms後にプレビュー更新
def on_text_change_debounced(self):
    parameters = self.get_parameters()
    self.parameter_changed.emit(parameters)  # プレビュー更新
```

### 日本語IME対応
```python
# QLineEditすべてに追加
input_field.setAttribute(Qt.WA_InputMethodEnabled, True)

# これにより、IME変換中のテキスト入力が適切に処理される
# 例：日本語「あ」→ ひらがな変換 → 確定
```

```python
# デフォルト: 500ms (バランスの良い設定)
```

## 🐛 トラブルシューティング

### 症状: まだテキスト入力が遅い/リセットされる

**解決策**:
1. アプリを再起動
2. デバウンス遅延を100msに設定してテスト
3. それでも問題がある場合、system logを確認

```bash
# macOS のログを確認
log stream --level debug | grep ATEM
```

### 症状: 日本語テキストが表示されない

**解決策**:
1. テンプレートのフォント名が正しいか確認
   - ✅ 正しい: "ヒラギノ角ゴシック"
   - ❌ 間違い: "Hiragino Sans" (英語表記)

2. macOSで日本語フォントがインストールされているか確認
   ```bash
   # Font Book.app で確認するか
   ls /Library/Fonts | grep -i hiragino
   ```

3. フォント名をドロップダウンから選択 (手入力ではなく)

### 症状: プレビューがまったく更新されない

**解決策**:
1. テキスト入力後、**500ms以上待つ**
2. マウスを動かす、別の要素をクリック、またはタブキーを押す
3. パラメータが入力されているか確認
